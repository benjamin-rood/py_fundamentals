# Lesson 7.3: What's Next — Carson's Playbook

**Duration:** 30 minutes (reading + reflection)

---

## What You Now Have

Over the past 8 weeks, you've built a foundation that most "learn to code" courses either skip entirely or bury under months of syntax drilling. Here's what you can do:

**Python literacy.** You can read Python code and understand what it does — variables, types, functions, conditionals, loops, lists, dicts, imports, error handling. When AI generates code, you can follow it.

**The object/reference model.** You know how Python actually works: objects in memory, variables as references, mutability, aliasing. This is what separates someone who can debug from someone who just stares at errors.

**Testing instincts.** You don't trust code until you've tested it — normal cases, boundaries, error cases. You know pytest, you've written real tests, and you've used tests to find bugs in AI-generated code.

**Git awareness.** You know what version control is, why it exists, and what's happening when AI agents say "I'll commit this" or "I'll create a branch." You've used the core commands.

**AI collaboration skills.** You can direct Claude Code with specific requests, review the output line by line, identify what's missing, write tests that expose the gaps, and improve the code until it's correct. This is the actual skill of coding in 2026.

**LLM understanding.** You know what the tool is doing under the hood — tokens, next-token prediction, why AI can be confidently wrong. You use AI tools with appropriate skepticism.

---

## Carson's Next Steps

Do these now. Don't take another course first. You're ready.

### 1. Think of something fun to build

Not useful — **fun**. A game. A joke app. A gift for someone. A tool that solves a tiny annoyance in your day. The point is to have a reason to keep going that isn't "I should learn more."

Carson's exact advice: "Think of something fun to build. Not something useful — something fun."

### 2. Build it in Replit with Claude Code

Open your Replit project (or create a new one). Start Claude Code. Describe what you want to build. Let it generate the first draft. Review it. Test it. Fix it. Repeat.

Talk to Claude Code like a colleague. When you're confused about a concept, ask it to explain — or open [claude.ai](https://claude.ai) in another tab for a side conversation while the build continues.

### 3. Deploy it

In Replit, deployment is essentially one click. Get something live on the internet. It doesn't have to be perfect. It doesn't have to be polished. It just has to be *yours* and *real*.

Showing someone a live link to something you built is more motivating than any certificate.

### 4. Learn just-in-time

You don't need to take another course. When you hit something you don't understand — a new library, a concept you haven't seen, a deployment issue — ask Claude Code. Or ask Claude in the chat. Or search Real Python.

That's the new learning model. You learn what you need, when you need it, from the tools that are right there.
